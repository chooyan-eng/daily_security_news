# Cursor AIコードエディタの重大脆弱性「DuneSlide」、プロンプトインジェクションでサンドボックス脱出

- **日付**: 2026-07-01
- **出典**: [The Hacker News](https://thehackernews.com/2026/07/critical-cursor-flaws-could-let-prompt.html)
- **トピック**: [Cursor AIコードエディタ サンドボックス脱出脆弱性「DuneSlide」（2026年）](../topics/cursor-ai-duneslide-2026.md)
- **分類**: 新規

## 概要

AIコードエディタ Cursor に2件の重大脆弱性「DuneSlide」（CVE-2026-50548、CVE-2026-50549、共にCVSS 9.8）が発見された。通常のプロンプトがAIエージェントのサンドボックスを脱出させ、開発者のマシン上で任意コマンドを実行可能にする。Cato AI Labs が発見し、Cursor 3.0（2026年4月2日リリース）で修正済み。

## 詳細

### DuneSlide の概要

Cursor は 2.x 系列以降、AIエージェントが発行するターミナルコマンドをデフォルトでサンドボックス内で実行する仕組みを持つ。DuneSlide は、間接的なプロンプトインジェクション（悪意あるファイルやWebページの内容にAIへの指示を埋め込む手法）を通じて、このサンドボックスから脱出する攻撃手法。Fortune 500企業の半数以上が Cursor を利用しているとされ、影響範囲は広い。

### CVE-2026-50548：作業ディレクトリの悪用

サンドボックスは、コマンドの作業フォルダへの書き込みを許可している。この作業フォルダは Cursor の `run_terminal_cmd` ツールの任意パラメータ `working_directory` で指定可能で、AIエージェントがこれをデフォルト以外のパスに設定すると、Cursor はそのパスを無条件に書き込み許可リストへ追加してしまう。インジェクションされた指示によってこのパラメータをプロジェクト外のシステムファイルへ向けさせ、サンドボックスヘルパー自体（macOS の場合 `/Applications/Cursor.app/Contents/Resources/app/resources/helpers/cursorsandbox`）を上書きすることで、以降のコマンドがサンドボックスなしで実行されるようになる。

### CVE-2026-50549：シンボリックリンク検証のフォールバック不備

Cursor は書き込み前に、シンボリックリンク（ショートカット）を解決して実際の書き込み先がプロジェクト内にあるかを確認する安全チェックを行う。しかしこのチェックの対象が存在しない場合や、攻撃者がパス中のフォルダへの読み取り権限を削除した場合など、検証に失敗するとチェックを諦め、シンボリックリンクが示すプロジェクト内パスをそのまま信頼してしまうフォールバック不備がある。

### 発見と対応

両脆弱性は Cato AI Labs によって発見され、CVSS 3.1で9.8（CVSS 4.0では9.3）と評価されている。Cursor 3.0（2026年4月2日リリース）以降で修正済みであり、それ以前の全バージョンが影響を受ける。エージェント型開発ツールにおけるプロンプトインジェクションからサンドボックス脱出、さらにリモートコード実行に至る攻撃チェーンの一例として、AIコーディングツール全般のセキュリティモデル見直しの必要性が指摘されている。

### 対策

- Cursor を 3.0 以降に速やかにアップデート
- AIエージェントに信頼できない外部コンテンツ（Webページ、リポジトリ内ファイル等）を処理させる際のプロンプトインジェクション対策の徹底
- `working_directory` のような任意パラメータを許可するツール呼び出しの監査
- エージェント型開発ツールの実行環境を最小権限の別コンテナやVMでさらに隔離する多層防御の検討
