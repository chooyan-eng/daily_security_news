# Microsoft AutoJack：悪意あるウェブページ1枚でAIエージェント経由のホストRCEが可能

- **日付**: 2026-06-20
- **出典**: [The Hacker News](https://thehackernews.com/2026/06/threatsday-bulletin-worm-code-leaked-ai.html)
- **トピック**: [Microsoft AutoJack AI エージェント RCE エクスプロイトチェーン（2026年）](../topics/microsoft-autojack-ai-agent-rce-2026.md)
- **分類**: 新規

## 概要

Microsoft の Defender Security Research Team が「AutoJack」と命名したエクスプロイトチェーンを公開した。Microsoft Research の AutoGen Studio（マルチエージェント AI プロトタイピング UI）の内蔵ウェブブラウジングエージェントを悪用し、悪意あるウェブページを閲覧させるだけでホストマシン上での任意コード実行（RCE）が可能となる。ユーザーの操作は URL の送信のみで済む。

## 詳細

### 概要

2026年6月18日、Microsoft Defender Security Research Team は AutoJack と命名されたエクスプロイトチェーンを Microsoft Security Blog で公開した。これは AI エージェントのウェブブラウジング機能を悪用してリモートコード実行を実現する新しい攻撃パターンを示したものである。

### 標的：AutoGen Studio

AutoJack の標的は Microsoft Research が開発するオープンソースのマルチエージェント AI プロトタイピングツール「AutoGen Studio」である。AutoGen Studio はユーザーが AI エージェントをビジュアルに構築・テストできるウェブ UI を提供しており、組み込みのウェブブラウジングエージェントを持つ。

### 3段階のエクスプロイトチェーン

AutoJack は 3 つの脆弱性を連鎖させる：

**段階1：オリジン許可リストのバイパス**
- AutoGen Studio のブラウジングエージェントは localhost として動作する
- localhost からのリクエストは信頼済みオリジンとして扱われ、オリジン検証をパスする
- 攻撃者が制御する外部ウェブページがこの信頼を悪用できる

**段階2：MCP エンドポイントの認証欠如**
- AutoGen Studio の MCP（Model Context Protocol）WebSocket エンドポイントに認証が存在しない
- エージェントが悪意あるページを閲覧すると、ページが未認証の MCP エンドポイントに直接アクセスできる

**段階3：安全でないパラメータ処理**
- MCP エンドポイントが攻撃者制御の値をシェルコマンドに直接渡す
- これによりコマンドインジェクションが成立し、ホストマシン上で任意コードが実行される

### 攻撃に必要なもの

攻撃者は以下のみを必要とする：
- 悪意あるコンテンツを持つ外部ウェブページ（通常の静的 HTML/JS）
- 被害者に URL を送信してエージェントに閲覧させること

ユーザーによる追加操作は不要であり（ゼロクリック RCE）、攻撃者は被害者のローカルホスト環境に一切アクセスできなくてもよい。

### 影響の範囲

AutoGen Studio をウェブブラウジングエージェント付きで実行しているユーザー・組織が対象となる。特に開発・研究・エンタープライズ環境での AI エージェント試験運用中のシステムが危険にさらされる。

### 修正状況

- **脆弱な MCP WebSocket サーフェスは PyPI リリース（pip インストール版）には含まれていない**: `pip install autogen-studio` でインストールした場合は影響を受けない
- **修正**: GitHub main の commit b047730（PR #7362）で修正済み
- **安定版リリース**: 修正済みの安定版はまだリリースされていない（2026年6月18日時点）
- **野生での悪用**: 確認されていない（研究目的の公開）

### 推奨対応

1. AutoGen Studio を GitHub からのソースビルドで使用している場合は、commit b047730 以降のバージョンに更新する
2. pip インストール版を使用している場合は現時点では影響なし（ただし安定版修正のリリースを待って更新）
3. AutoGen Studio のブラウジングエージェントへの外部 URL 送信に注意する
4. ファイアウォールまたはネットワークポリシーで AutoGen Studio の MCP ポートへの外部アクセスをブロックする

### セキュリティへの示唆

AutoJack は AI エージェントのブラウジング機能が持つ本質的なリスクを示している。エージェントがローカルで動作し、ウェブコンテンツにアクセスできる場合、そのエージェント自体が攻撃のベクターとなりうる。AI エージェントのセキュリティ設計では、ブラウジング機能の権限分離と入力検証が不可欠である。
