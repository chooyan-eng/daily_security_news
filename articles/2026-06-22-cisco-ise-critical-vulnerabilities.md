# Cisco ISE クリティカル脆弱性（CVE-2026-20181/20190）：認証済みRCEと認証情報漏洩

- **日付**: 2026-06-22
- **出典**: [Cybersecurity News](https://cybersecuritynews.com/cisco-ise-rec-vulnerability/)
- **トピック**: [Cisco Catalyst SD-WAN CVE-2026-20262 ゼロデイ悪用（2026年6月）](../topics/cisco-sdwan-cve-2026-20262.md)
- **分類**: 関連

## 概要

Cisco は2026年6月17日付けのセキュリティアドバイザリ（cisco-sa-ise-multi-G5WP8vv）で、Cisco Identity Services Engine（ISE）および ISE Passive Identity Connector（ISE-PIC）に存在する2件のクリティカル脆弱性を公開した。CVE-2026-20181（RCE、CVSS 9.1）および CVE-2026-20190（情報漏洩）で、構成に関わらず影響を受ける。現時点での積極的な悪用は未確認だが、ネットワーク制御・認証の中枢を担う製品だけに、ゼロデイ悪用への転化リスクが高い。

## 詳細

### 対象製品

- **Cisco Identity Services Engine (ISE)** — ネットワークアクセス制御（NAC）・ポリシー管理プラットフォーム
- **Cisco ISE Passive Identity Connector (ISE-PIC)** — Active Directory 等との ID 連携コンポーネント

### CVE-2026-20181 — リモートコード実行（CVSS 9.1）

**脆弱性の概要：**
ユーザー供給入力の不適切な検証に起因する RCE 脆弱性。管理者権限を持つ認証済み攻撃者が、細工した HTTP リクエストを送信することで基盤となる OS 上で任意コマンドを実行できる。

**攻撃シナリオ：**
1. 攻撃者がユーザーレベルのアクセスを取得（フィッシング・パスワードスプレー等）
2. CVE-2026-20181 を悪用して root 権限へ特権昇格
3. デバイスの完全制御を取得

**影響範囲：** 構成に関わらず全展開に影響

### CVE-2026-20190 — 機密情報漏洩

**脆弱性の概要：**
不適切な認可チェックにより、未認証のリモート攻撃者が API エンドポイントへのアクセスを通じて機密データを取得できる。

**漏洩する情報：** ハッシュ化されたクレデンシャルを含む機密情報

**攻撃の前提条件：** 認証不要（ネットワーク到達性のみ）

### 修正バージョン

| ISE バージョン | 修正パッチ | 提供時期 |
|--------------|----------|---------|
| ISE 3.3 | Patch 11 | 提供中 |
| ISE 3.4 | Patch 6 | 提供中 |
| ISE 3.5 | Patch 4 | 2026年8月予定 |

- ISE 3.5 より前のバージョン（3.0, 3.1, 3.2）はサポート終了済みのため、サポート対象バージョンへの移行が必要
- 回避策（ワークアラウンド）は存在しない。パッチ適用が唯一の有効な緩和策

### セキュリティへの影響

Cisco ISE は多くの企業・政府機関でゼロトラストネットワークアクセス（ZTNA）の根幹を担う製品であり、ISE が侵害されると：

- ネットワーク全体のアクセス制御が無効化される危険がある
- Active Directory / LDAP の認証情報が取得される
- VPN・Wi-Fi・有線ネットワークへの不正アクセスの踏み台となる
- ゼロトラストセキュリティポリシー全体の崩壊につながる

### 既存 Cisco 脆弱性との関連

2026年6月には Cisco 製品に連続して重大脆弱性が公開されており、Cisco Catalyst SD-WAN の CVE-2026-20262（パストラバーサル・ファイル書き込み）との関連性は直接はないが、Cisco ネットワーク製品全体を標的とした系統的な調査・攻撃活動の一環とみる研究者もいる。
