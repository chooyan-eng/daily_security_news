# Atomic Arch：1,500件以上のAURパッケージにeBPFルートキットを注入するサプライチェーン攻撃

- **日付**: 2026-06-20
- **出典**: [SecurityWeek](https://www.securityweek.com/atomic-arch-supply-chain-attack-hits-1500-aur-packages/)
- **トピック**: [Atomic Arch AUR サプライチェーン攻撃（2026年6月）](../topics/atomic-arch-aur-supply-chain-2026.md)
- **分類**: 新規

## 概要

2026年6月11〜12日に公開された「Atomic Arch」キャンペーンは、Arch Linux の Arch User Repository（AUR）の孤立パッケージ引き継ぎプロセスを悪用し、約1,500件のパッケージに悪意あるコードを注入した。Rust 製のインフォスティーラーと eBPF ベースのルートキット機能を組み合わせ、開発者ワークステーションや CI/CD 環境を狙う高度なサプライチェーン攻撃である。

## 詳細

### インシデント概要

2026年6月11〜12日、大規模な AUR（Arch User Repository）サプライチェーン攻撃が公開された。「Atomic Arch」と命名されたこのキャンペーンは、AUR のパッケージ孤立化・引き継ぎ（adoption）プロセスを悪用したものである。

### AUR のパッケージ引き継ぎメカニズムの悪用

AUR では、メンテナーが更新を停止したパッケージが「孤立（orphaned）」状態になると、コミュニティメンバーが引き継ぐことができる。攻撃者はこのシステムを悪用し、多数の孤立パッケージを組織的に引き継いだ上で、ビルドスクリプト（PKGBUILD）に悪意あるコードを注入した。

### 攻撃の展開

**第1波（6月11日）**：
- 約408件の孤立パッケージを引き継ぎ
- `atomic-lockfile` という悪意ある npm パッケージを依存関係として追加するコードを PKGBUILD に注入
- パッケージビルド（`makepkg`）時に自動的に悪意あるコードが実行される

**第2波（6月12日、24時間以内）**：
- 配布ベクターを切り替え：`bun` ランタイム経由でインストールされる `js-digest` npm パッケージを使用
- 合計侵害パッケージ数が1,500件以上に拡大

### マルウェアの技術的機能

注入されたマルウェアは Rust で実装されており、以下の機能を持つ：

**情報窃取機能**：
- 開発者ワークステーション上の認証情報・シークレットの収集
- CI/CD 環境の環境変数（API キー・トークン等）の窃取
- SSH キー・GPG キーの収集
- ブラウザ保存パスワードの窃取

**eBPF ベースのルートキット機能**（root 権限取得時）：
- eBPF（Extended Berkeley Packet Filter）を使用したカーネルレベルの監視回避
- プロセス・ネットワーク通信の隠蔽
- ファイルシステムレベルのステルス操作
- 通常のプロセスリストやネットワーク監視ツールからの隠蔽

### 標的

主な標的は Arch Linux または Arch ベースのディストリビューション（Manjaro・EndeavourOS 等）を使用する：
- ソフトウェア開発者
- セキュリティ研究者
- CI/CD パイプライン（自動化ビルド環境）

開発者環境を標的にすることで、ダウンストリームの組織・プロジェクトへの二次攻撃（サプライチェーンの多段階汚染）が可能となる。

### 対応状況

Arch Linux のメンテナーチームは侵害されたパッケージを迅速に AUR から削除した。コミュニティが検出ツール（`aur-malware-check`）を GitHub で公開しており、侵害チェックが可能。

### 推奨対応

1. 2026年6月11〜12日の間に AUR パッケージをビルド・インストールした場合は侵害を前提として対応する
2. GitHub 上の `aur-malware-check` ツールを使用して影響を確認する
3. 認証情報（SSH・GPG・API キー等）を完全にリセットする
4. CI/CD 環境の環境変数をすべて再生成する
5. AUR の孤立パッケージの自動インストールを避け、信頼できるメンテナーのパッケージのみを使用する
