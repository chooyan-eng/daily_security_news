# Cordyceps：CI/CDワークフローの設定不備を悪用した大規模サプライチェーンリスク（Microsoft・Google・Apache等300以上のリポジトリに影響）

- **日付**: 2026-06-24
- **出典**: [The Hacker News](https://thehackernews.com/2026/06/cordyceps-cicd-flaws-expose-300-github.html)
- **トピック**: [Cordyceps CI/CDワークフロー脆弱性パターン（2026年）](../topics/cordyceps-cicd-vulnerability-2026.md)
- **分類**: 新規

## 概要

Novee Security が「Cordyceps」と命名した CI/CD ワークフロー設定不備パターンを発見。GitHub Actions の過剰権限 PR ワークフローを悪用することで、未認証の第三者がリポジトリを乗っ取り、コード実行・クレデンシャル窃取・サプライチェーン汚染が可能。Microsoft、Google、Apache、Cloudflare を含む大規模組織の300以上のリポジトリが完全に悪用可能な状態と判明した。

## 詳細

### 脆弱性パターンの概要（Cordyceps）

「Cordyceps」は特定の単一 CVE ではなく、GitHub Actions における CI/CD ワークフロー設定の構造的な不備パターン。ノベエセキュリティ（Novee Security）の研究者 Elad Meged が発見し、2026年6月に公開した。

**問題の核心：** プルリクエスト（PR）から起動されるワークフローが、本来必要以上の権限（シークレット・デプロイキー・クラウド認証情報・リリース署名鍵へのアクセス権）を持っている。

### 攻撃手法

1. 攻撃者は無料の GitHub アカウントを作成するだけでよく（org メンバーシップや特別な権限は不要）
2. 標的リポジトリに悪意ある PR を送信する
3. Cordyceps パターンを持つワークフローが PR の内容をそのまま実行してしまう
4. 結果として：
   - 攻撃者制御のコードがビルドパイプラインで実行される
   - CI/CD が保持するシークレット（API キー、クラウド資格情報、署名鍵等）が窃取される
   - 悪意あるコードをリリースアーティファクトや npm/PyPI パッケージに混入できる

### 調査結果

- 調査対象: ~30,000 の高影響リポジトリ（フォロワー数・ダウンロード数が多い OSS 等）
- 完全に悪用可能と判定されたリポジトリ: 300件以上
- 影響を受ける組織例: Microsoft、Google、Apache、Cloudflare など

### AI コーディングエージェントが拡大要因

特に懸念される点は、AI コーディングエージェント（GitHub Copilot や他の LLM ベースツール）が CI/CD 設定ファイルを自動生成する際に、同じ脆弱なパターンを繰り返し出力する傾向があること。開発者が AI に任せて作成した .github/workflows/ ファイルが、意図せず Cordyceps パターンを含む可能性がある。

### 既存のサプライチェーン攻撃との違い

従来のサプライチェーン攻撃（npm パッケージ汚染等）は悪意あるパッケージを注入するものが多かったが、Cordyceps は**CI/CD インフラ自体**を乗っ取ることで、正規のビルドプロセス経由でマルウェア配布や秘密情報窃取が行える点で深刻度が高い。

### 修正方法

- `pull_request_target` トリガーには細心の注意を払い、PR からの入力をシェルコマンドに渡さない
- ワークフローの権限スコープを最小限に制限（`permissions: read-all` など）
- `GITHUB_TOKEN` の権限を明示的に最小化
- Novee Security が公開したスキャナーツールで自リポジトリを確認する
