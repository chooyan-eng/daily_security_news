# Azure CLIを狙った大規模パスワードスプレー攻撃、8100万回超の試行で78アカウント侵害

- **日付**: 2026-07-01
- **出典**: [The Hacker News](https://thehackernews.com/2026/07/azure-cli-password-spray-hits-at-least.html)
- **トピック**: [Azure CLI パスワードスプレー攻撃「LSHIY」キャンペーン（2026年）](../topics/azure-cli-password-spray-lshiy-2026.md)
- **分類**: 新規

## 概要

セキュリティ企業 Huntress は、Microsoft の Azure CLI（コマンドラインインターフェース）を標的とした大規模かつ自動化されたパスワードスプレー攻撃を確認した。2026年6月12日から26日の間に8100万回超のログイン試行が行われ、64組織にまたがる少なくとも78のMicrosoftアカウントが侵害された。攻撃はROPC OAuth機構を悪用し、多要素認証（MFA）を回避している。

## 詳細

### 攻撃規模と時期

Huntress によると、攻撃者は2026年6月12日から26日までの2週間で8100万回を超えるログイン試行を実行し、64組織にまたがる少なくとも78のMicrosoftアカウントの侵害に成功した。

### 攻撃元

活動はインターネットインフラ事業者 LSHIY LLC（AS32167）が管理するIPv6アドレス範囲（2a0a:d683::/32）から発生している。攻撃者は、過去のデータ侵害で流出した既知のユーザー名・パスワードの組み合わせを用い、Azure CLI経由での認証を試みた。

### MFA回避の技術的仕組み

有効な認証情報の組み合わせが見つかると、攻撃者は ROPC（Resource Owner Password Credentials）というOAuth認証方式を用いてログインする。多くの環境で、条件付きアクセスポリシーの設定不備により、この方式は多要素認証（MFA）をバイパスしてしまう。ROPCはユーザー名とパスワードを直接引き渡して認証する古い方式であり、モダン認証フローでMFAを強制する仕組みが適用されないケースが多い。

### 傾向

Huntress は、過去6ヶ月間で顧客基盤全体における認証情報スプレー攻撃の量が155倍以上に増加していると報告している。レガシーな認証プロトコル（ROPCなど）が依然として多くの組織で有効化されたままであることが、こうした攻撃の温床となっている。

### 対策

- Azure AD/Entra ID における ROPC を含むレガシー認証プロトコルの無効化
- 条件付きアクセスポリシーでMFAを確実に強制する設定への見直し
- LSHIY LLC（AS32167）関連のIPv6レンジからのアクセスの監視・ブロック
- 過去の侵害で流出した認証情報の再利用がないか、パスワード漏洩チェックサービス等で確認
- Azure CLI 経由のサインインログの異常検知強化
