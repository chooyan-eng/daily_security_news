# 282件のiOS向けAIチャットアプリがLLM APIキー・プロキシアクセスを通信トラフィック上に漏洩

- **日付**: 2026-06-30
- **出典**: [The Hacker News](https://thehackernews.com/2026/06/282-ios-apps-found-leaking-llm-api-keys.html)
- **トピック**: [iOS AIアプリ LLM APIキー漏洩問題（2026年）](../topics/ios-ai-apps-llm-key-leak-2026.md)
- **分類**: 新規

## 概要

Wake Forest大学の研究チームがiPhone向けAIチャットアプリ444本を調査した結果、282本（約64%）が通信トラフィック上でLLM APIへの有料アクセス権を漏洩していることが判明した。JWTトークン漏洩、未認証バックエンドプロキシアクセス、平文APIキー送信が主な原因で、攻撃者による不正利用で1日あたり4万6000ドル超の課金被害が生じうると試算されている。

## 詳細

### 調査概要

研究チームは独自開発したトラフィック解析ツール「LLMKeyLens」を用い、ジェイルブレイクやアプリの分解を行わずにアプリの通信を観測するだけでAPI認証情報を抽出できることを実証した。対象は少なくとも10社のAIプロバイダーにまたがり、OpenAIへの漏洩が最多だった。

### 漏洩パターンの内訳

- **JWTトークン漏洩**: 48%
- **未認証バックエンドプロキシアクセス**: 33%
- **平文APIキー送信**: 19%

### 影響範囲

漏洩は生産性ツール、健康・フィットネス、エンターテインメントなど13のアプリカテゴリーに及ぶ。生産性アプリが最も件数が多かった一方、漏洩率が最も高かったのは健康・フィットネスアプリだった。対象の多くは小規模アプリだが、なかには200万件超のユーザー評価を持つ人気アプリも含まれていた。

### 金銭的リスク

研究者の試算では、漏洩した認証情報が悪用された最悪のケースで1日あたり4万6000ドル以上のAI利用料金が不正に発生しうるとされる。

### 開発者への通知と改善状況

研究チームは漏洩を確認した282件すべての開発者に通知し、3か月間の猶予期間を設けて追跡調査を行った。その結果、明確に修正が確認できたのはわずか28%にとどまり、23%は依然として漏洩したアクセスが有効なまま放置されていた。

### 推奨対応（開発者向け）

- APIキー・トークンをクライアント側（モバイルアプリ内）に直接埋め込まない
- バックエンドプロキシには必ず認証・レート制限を実装
- 通信内容の暗号化に加え、証明書ピンニング等でMITM経由の傍受リスクを低減
- 定期的な自社アプリの通信トラフィック監査の実施
