# Google Vertex AI SDK「Pickle in the Middle」：バケット乗っ取りでAIモデル汚染とRCEが可能

- **日付**: 2026-06-20
- **出典**: [The Hacker News](https://thehackernews.com/2026/06/google-vertex-ai-sdk-flaw-let-attackers.html)
- **トピック**: [Google Vertex AI SDK バケット乗っ取り脆弱性（2026年）](../topics/google-vertex-ai-sdk-pickle-2026.md)
- **分類**: 新規

## 概要

Palo Alto Networks Unit 42 が「Pickle in the Middle」と命名した Google Cloud Vertex AI SDK for Python の脆弱性により、攻撃者は被害者のプロジェクトへのアクセスなしに ML モデルアップロードを乗っ取り、Google の本番サービスインフラ上でコードを実行できることが判明した。Google はバグバウンティプログラムを通じた報告を受け、v1.148.0 で完全修正済み。

## 詳細

### 脆弱性の概要

2026年6月16日、Palo Alto Networks の Unit 42 チームが Google Cloud Vertex AI SDK for Python に存在する重大な脆弱性を公開した。攻撃手法は「Pickle in the Middle」と命名され、Google のバグバウンティプログラムを通じて事前に報告・修正されている。

### 攻撃の仕組み（バケット乗っ取り）

この脆弱性は、SDK がモデルアップロード時に使用する一時 Cloud Storage バケット名の生成ロジックに起因する：

1. **予測可能なバケット名の生成**: ユーザーがバケットを指定しない場合、SDK はプロジェクト ID とリージョンに基づいてバケット名を自動生成する。このプロジェクト ID は多くの場合公開されている
2. **バケットの先取り（Bucket Squatting）**: 攻撃者は被害者のプロジェクト ID を知るだけで、被害者が使用するバケット名を予測し、自身の Google Cloud プロジェクトでそのバケットを事前に作成しておける
3. **モデルファイルの差し替え**: 被害者の SDK が攻撃者のバケットにモデルファイルをアップロードすると、攻撃者はファイルを悪意あるモデルに差し替える
4. **サービスインフラ上での RCE**: Vertex AI が差し替えられたモデルをロードすると、pickle/joblib ライブラリのデシリアライズ処理が攻撃者のコードを実行する

### なぜ危険か：pickle の設計上の特性

Python の機械学習エコシステムでは、モデルの保存に pickle や joblib が広く使われている。これらのフォーマットはデシリアライズ時に任意コードを実行するよう設計されており、信頼できないソースからのモデルロードが本質的に危険である。

攻撃者のコードは Google の本番サービスコンテナ内で実行されるため、Google のインフラへの深刻な侵害が可能となる。

### 攻撃に必要な条件

- 被害者の Google Cloud プロジェクト ID（多くの場合公開済み）
- 攻撃者自身の Google Cloud プロジェクト（無料枠で作成可能）
- フィッシングなし、パスワードなし、被害者プロジェクトへのアクセスなし

### Google の対応

- **v1.144.0（2026年3月31日）**: バケット名にランダムな UUID4 を追加する初期修正
- **v1.148.0（2026年4月15日）**: `Model.upload()` にバケット所有権検証を追加してバケット乗っ取りを完全にブロックする最終修正

### 影響の範囲

脆弱なバージョン（v1.144.0 より前）を使用して ML モデルをアップロードした組織は、モデルが汚染（モデルポイズニング）されている可能性がある。また、Google のサービスインフラへの RCE リスクもあったが、現在は修正済み。

### 推奨対応

1. `google-cloud-aiplatform` SDK を v1.148.0 以降に更新する
2. バケットを明示的に指定してモデルアップロードを実行する（自動生成バケットを使用しない）
3. Vertex AI を通じてロードされる ML モデルの整合性を検証する（ハッシュチェック等）
4. 既存のモデルが汚染されていないかを確認する
