# OpenClaw AIエージェントにプロンプトインジェクションとデータ窃取を可能にする複数脆弱性

- **日付**: 2026-06-16
- **出典**: [The Hacker News](https://thehackernews.com/2026/05/four-openclaw-flaws-enable-data-theft.html)
- **トピック**: [OpenClaw AIエージェントのセキュリティ脆弱性](../topics/openclaw-ai-security.md)
- **分類**: 新規

## 概要

人気の自己ホスト型AIエージェント「OpenClaw」に複数の深刻なセキュリティ脆弱性が発見された。攻撃者が共有連絡先、vCard、位置情報ピンなどに隠したプロンプトインジェクション命令を通じてエージェントを操作し、任意コードの実行や機密データの窃取が可能。構造的な問題のため根本解決が困難とされる。

## 詳細

### 背景

OpenClawは自己ホスト可能なAIエージェントプラットフォームで、タスクの自動化、データ処理、外部サービスとの連携などに広く利用されている。2026年初頭のセキュリティ監査では512件の脆弱性が発見され、うち8件がCriticalに分類された。

### 主要な脆弱性

#### 間接プロンプトインジェクション（Indirect Prompt Injection）
- 攻撃者がエージェントの処理対象データ（vCard、共有連絡先、位置情報ピンなど）に悪意あるプロンプト命令を埋め込む
- エージェントがその命令を正規の指示と解釈し、機密情報の漏洩や意図しないアクションを実行
- ImpervaaのセキュリティチームがOpenClawのvCard・連絡先データ処理経由での攻撃を実証

#### ClawJacked（ローカルインスタンスハイジャック）
- 悪意あるWebサイトがブルートフォースにより、ローカルで動作するOpenClawインスタンスをハイジャック可能
- Oasis Securityが発見、エージェントの自律性を悪用して静かにデータを外部流出させることが可能
- OpenClawはバージョン2026.2.26（2026年2月26日リリース）でパッチを適用

#### ClawHubのマルウェアスキル
- セキュリティ監査により、ClawHub（スキルマーケットプレイス）に登録されたスキルの最大12%が悪意あるもの（クレデンシャルスティーラー、ガイダンスインジェクションなど）と判明

### 構造的な問題

LLMは命令とデータを同一のコンテキストウィンドウで処理するため、プロンプトインジェクション脆弱性は本質的にアーキテクチャ上の問題。完全な解決策は現時点では存在せず、緩和策として以下が推奨される：

- 最小権限の原則を徹底（エージェントに必要最低限のアクセス権のみ付与）
- コンテナ分離によるサンドボックス化
- 不正アクションをブロックするランタイムセキュリティツールの導入
- 信頼できるソースからのスキルのみをインストール
