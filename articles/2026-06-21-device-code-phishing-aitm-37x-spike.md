# デバイスコードフィッシング（AiTM）が2026年6月に37倍急増 — MFA を無効化するクリミナルコモディティ化

- **日付**: 2026-06-21
- **出典**: [Push Security Blog](https://pushsecurity.com/blog/device-code-phishing) / [BleepingComputer](https://www.bleepingcomputer.com/news/security/) / [Microsoft Security Blog](https://www.microsoft.com/en-us/security/blog/2026/05/04/breaking-the-code-multi-stage-code-of-conduct-phishing-campaign-leads-to-aitm-token-compromise/)
- **トピック**: [デバイスコードフィッシング / AiTM セッショントークン窃取（2026年）](../topics/device-code-phishing-aitm-2026.md)
- **分類**: 新規

## 概要

OAuth 2.0 デバイス認証フロー（Device Authorization Grant）を悪用してMFAを回避するデバイスコードフィッシング攻撃が、2026年6月に37倍の検出スパイクを記録。18種類のフィッシングキットが登場し、すべての主要 AiTM プラットフォームがこの手法を採用したことで、かつてスパイ活動専用だった技術が月250ドルのサブスクで購入できるサイバー犯罪コモディティとなった。Microsoft 365、Azure AD をはじめとするクラウドプラットフォームが標的。

## 詳細

### デバイスコードフィッシングとは

OAuth 2.0 の Device Authorization Grant は、入力機能が限られたデバイス（テレビ、IoT デバイスなど）向けに設計された認証フロー。攻撃者はこのフローを悪用し、

1. 攻撃者が正規の OAuth フローを開始し、デバイスコード（例: `ABCDE-12345`）と認証 URL を取得
2. フィッシングメールでコードを被害者に送付（「セキュリティ更新が必要」「重要文書を確認してください」等の口実）
3. 被害者が正規の Microsoft/Google サイトでコードを入力・承認
4. 攻撃者のデバイスにアクセストークン + リフレッシュトークンが発行される

**結果**: パスワードや MFA コードを一切盗まず、有効なセッショントークンを直接取得。

### 2026年の爆発的拡大

| 指標 | 数値 |
|------|------|
| 6月の検出数増加率 | 37倍 |
| 確認されたフィッシングキット数 | 18種類 |
| Telegram での販売価格 | 月約 $250〜 |
| 主要 AiTM プラットフォームの対応 | すべてが採用済み |

### 主要キャンペーン事例

**Microsoft 大規模キャンペーン（2026年4月）**  
4月14〜16日の72時間に26カ国・1万3000組織超・3万5000人以上を標的にしたスピアフィッシングキャンペーンを Microsoft が観測。標的の92%は米国の組織。

**FBI 警告（2026年5月21日）**  
FBI が PSA を発出し、Microsoft 365 アカウントの OAuth トークン窃取ツールが Telegram で販売されていると警告。パスワードを一切触れずにアカウントを乗っ取るため、従来の認証情報盗難防御が無効。

### なぜ従来の防御が効かないのか

- **強力なパスワード**: 無効（パスワードを使わない）
- **TOTP/SMS MFA**: 無効（認証フロー自体を悪用）
- **パスキー**: 無効（デバイス認証レイヤーをバイパス）
- **フィッシング対策学習**: 効果が限定的（正規サイト上での操作のため）

### 対策

- **デバイスコードフローを無効化**または使用するデバイス/アプリを厳格に制限（Microsoft Entra ID ではポリシーで設定可能）
- **条件付きアクセスポリシー**: 認可されていない場所・デバイスからのトークン要求を拒否
- **継続的アクセス評価（CAE）**: セッション中の異常を検知し強制失効
- **ユーザー教育**: デバイスコード入力を求めるメールは一切信用しない
