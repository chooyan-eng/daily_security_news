# ServiceNow APIの認証バイパス脆弱性でエンタープライズ顧客データが流出

- **日付**: 2026-06-16
- **出典**: [BleepingComputer](https://www.bleepingcomputer.com/news/security/servicenow-discloses-security-incident-exposing-customer-data/)
- **トピック**: [ServiceNow APIインシデント2026](../topics/servicenow-api-incident-2026.md)
- **分類**: 新規

## 概要

エンタープライズSaaSプロバイダーのServiceNowは、顧客インスタンスデータへの不正アクセスを引き起こしたAPIの認証バイパス脆弱性を2026年6月5日に修正した。攻撃者は6月2〜3日にわたり、認証なしでAPIにアクセスし、ITチケット、従業員情報、内部ドキュメントなどを含む顧客データを窃取した可能性がある。

## 詳細

### インシデントの概要

ServiceNowのScripted REST Resourceにおける設定ミスが原因で、特定のAPIエンドポイントが認証なしでアクセス可能な状態になっていた。問題のエンドポイント`/api/now/related_list_edit/create`が`requires_authentication=false`の設定になっており、認証不要でHTTPリクエストが送信可能だった。

### 影響を受けるデータ

ServiceNowの顧客インスタンスには以下のようなデータが保存されており、攻撃者がアクセスした可能性がある：

- ITサポートチケット
- 従業員レコード
- 内部ナレッジベース記事
- サポートケース内に含まれる認証情報・APIトークン・シークレット

### タイムライン

- **2026年6月2〜3日**：不正アクセスが発生
- **2026年6月5日**：ServiceNowがセキュリティアップデートを展開して脆弱なエンドポイントを修正
- **2026年6月10日**：TechTimesが報道（ServiceNowが当初のアドバイザリをゲートで隠し、顧客への通知が遅れた問題も指摘）

### 影響範囲

主に「Australia」プラットフォームリリースの顧客、または旧リリース上で特定の設定変更を行った顧客が影響を受けた。ServiceNowは影響を受けた顧客にサポートポータルを通じて通知を開始している。

### 注目ポイント

- CVEは記事執筆時点で未割り当て（ServiceNowが割り当て要否を検討中）
- アドバイザリがゲート（ログイン必須）後ろに隠され、顧客が即座に情報にアクセスできなかったことへの批判も
- エンタープライズSaaSのAPIセキュリティ設定の重要性を改めて示したインシデント

### 推奨対応

- ServiceNowのサポートポータルで自インスタンスへの不審なアクセスがないか確認
- ServiceNowから届く通知メールを確認し、影響を受けているかを把握
- インスタンスのAPIエンドポイント設定を監査し、不要な匿名アクセスが許可されていないかを確認
