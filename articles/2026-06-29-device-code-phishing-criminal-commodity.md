# デバイスコードフィッシング 完全なコモディティ化 — 18キット・37倍スパイク・全主要AiTMベンダーが採用

- **日付**: 2026-06-29
- **出典**: [BleepingComputer](https://www.bleepingcomputer.com/news/security/) / [The Hacker News](https://thehackernews.com/)
- **トピック**: [デバイスコードフィッシング / AiTM セッショントークン窃取（2026年）](../topics/device-code-phishing-aitm-2026.md)
- **分類**: 続報

## 概要

2026年6月下旬の最新レポートにより、OAuth 2.0 デバイス認証フロー（Device Authorization Grant）を悪用するAiTM攻撃の「犯罪エコシステムへの完全統合」が改めて確認された。検出数は37倍のスパイクを記録し、18種類のフィッシングキットが確認され、国家諜報機関専用だった高度な手法が月$250程度のサブスクリプションで誰でも利用できるコモディティとなった。すべての主要AiTMベンダーがこの攻撃タイプをプラットフォームに追加したことで、攻撃のスケールアップが容易になっている。

## 詳細

### デバイスコードフィッシングの現状

当初は国家レベルのスパイ活動に使われていたデバイスコードフィッシング（Device Code Phishing / OAuth Device Authorization Grant Attack）が、2026年上半期に犯罪エコシステムへ完全に取り込まれた。

**定量的指標**:
- **フィッシングキット数**: 18種類が確認済み（2026年第2四半期時点）
- **検出スパイク**: 前年同期比 **37倍**
- **Telegrmでの販売価格**: 月額 **約 $250〜**
- **対応済みAiTMプラットフォーム**: すべての主要ベンダーが対応追加

### 攻撃の仕組みのおさらい

OAuth 2.0 Device Authorization Grant フローを悪用：

1. 攻撃者が Microsoft/Google 等の認証サービスに向けてデバイスコード要求を開始
2. 生成されたデバイスコードと認証URL（`microsoft.com/devicelogin` 等）を被害者へ送信（フィッシング）
3. 被害者が正規のドメインで認証を完了
4. 攻撃者がアクセストークンとリフレッシュトークンを取得
5. パスワードもMFAも突破され、セッションを完全に制御

**対策として無効なもの**: パスワード強化、TOTP型MFA、SMS型MFA、パスキー

### 2026年後半の新動向

#### AiTM プラットフォームへの統合

すべての主要AiTMフィッシングキットベンダーがデバイスコード攻撃モジュールを追加したことで：

- 攻撃者がゼロから実装不要で高度な攻撃が可能
- 技術スキルの低い犯罪者でも容易に展開可能
- 攻撃のスケールアップが低コストで実現

#### 国家系グループと犯罪グループの手法共有

かつては APT グループ（ロシア・北朝鮮等の国家系）が使用していた手法が：

- Telegram マーケットプレイスでのキット販売
- MaaS（Malware-as-a-Service）モデルへの組み込み
- サイバー犯罪フォーラムでの手法共有

といった経路で広範に普及した。ロシア諜報機関による Signal バックアップキーフィッシングとは攻撃対象は異なるが、「公式サービスを装ってセッション/認証情報を窃取する」という本質的な手法が国家系から犯罪エコシステムへ移転するパターンの典型例となっている。

### 有効な対策

1. **FIDO2 / WebAuthn ハードウェアセキュリティキー**: デバイスコードフローでのセッショントークン窃取に対して有効な数少ない対策
2. **条件付きアクセスポリシー**: デバイスコードフローの使用自体をブロックまたは制限
3. **異常なOAuthトークン発行の監視**: 通常使われないデバイスコードフロー由来のトークンを検出
4. **ユーザー教育**: デバイスコード入力を要求する予期しないメッセージへの警戒
