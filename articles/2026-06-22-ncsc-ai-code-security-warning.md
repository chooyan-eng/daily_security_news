# 英国NCSC「AIが生成したコードは慎重なレビューなしでは重大なセキュリティリスク」と警告

- **日付**: 2026-06-22
- **出典**: [Cybernews](https://cybernews.com/ai-news/britain-cyber-ai-code-security-disasters/)
- **トピック**: [英国NCSCのAIコード生成セキュリティリスク警告（2026年）](../topics/ncsc-ai-code-security-2026.md)
- **分類**: 新規

## 概要

英国の国家サイバーセキュリティセンター（NCSC）は2026年6月22日、AIによって生成されたコードを適切な精査なしに利用することが深刻なセキュリティリスクをもたらすと警告した。「バイブコーディング（vibe coding）」と呼ばれる、AIの提案をそのまま採用する開発スタイルは低リスクプロジェクトにとどめ、重要システムでは徹底的なレビュー・テスト・監査を行うべきと指摘している。

## 詳細

### 警告の背景

AIコーディングアシスタント（GitHub Copilot、Claude Code、Cursor 等）の急速な普及に伴い、開発者がAIの生成コードをそのまま採用する「バイブコーディング」が一般化しつつある。これに伴い、AIが生成したコードに含まれる脆弱性や、検査しにくい複雑なコードベースが本番環境に混入するリスクが高まっていると NCSCは指摘する。

### NCSCが指摘する主なリスク

**1. セキュリティ脆弱性の混入**
AIモデルは学習データに含まれる既知の脆弱性パターンを反映したコードを生成することがある。SQLインジェクション、バッファオーバーフロー、不適切な認証処理などが代表例。

**2. 監査困難なコードベース**
AIが生成したコードは人間の書き方とは異なる構造になることがあり、セキュリティチームによるコードレビューや監査が困難になる場合がある。

**3. 過剰な信頼（Over-reliance）**
開発者がAIの出力を過信し、動作テストのみ行って詳細なセキュリティ検査をスキップするリスクがある。

**4. 将来の潜在性に対する過信**
現時点のAIの実力を超えた信頼を置くことで、不適切なセキュリティ基準が生産環境に組み込まれる危険がある。

### NCSCのガイダンス

NCSCは以下のアプローチを推奨している：

- **リスクベースのアプローチ**：プロジェクトのリスクレベルに応じてAI活用の深度を調整する
- **徹底したテストと監査**：AI生成コードを本番環境に適用する前に、テスト・監査・セキュリティレビューを実施する
- **コードを理解した上での採用**：AIが生成したコードを開発者自身が理解していることを確認する
- **クリティカルシステムへの適用は慎重に**：重要インフラや機密性の高いシステムへのAI生成コード適用は特別な慎重さが必要

### 関連動向

NCSCはAIを活用した脆弱性発見（AI-aided vulnerability discovery）についても別途ガイダンスを発行しており、AI活用前にセキュリティ・法的・運用・リソース面でのリスクを考慮するよう企業に促している。

### 業界への示唆

Anthropicの Claude Code に関しては、Check Point Research が2026年2〜6月にかけて複数の重大脆弱性（CVE-2025-59536, CVE-2026-21852 等）を発見しており、AIコーディングツール自体がサイバー攻撃の新たな攻撃面になりつつあることを示している。NCSCの警告はこうした背景を踏まえたものでもある。
